## 📖 Comprehensive Roadmap & Developer Guide for Globally Linked NOVA CLI

## 🔗 1. تشغيل المنصة عالمياً بعد ربطها (npm link)
عند تنفيذ `npm link` داخل مجلد المنصة الرئيسي:
1. يتم ربط حزمة `nova-cli` بمجلد الـ `node_modules` العالمي الخاص بـ `npm`.
2. يتم تسجيل الأمر `nova` في نظام التشغيل لديك (سواء Windows PowerShell أو Linux/macOS Terminal).

### الأوامر الأساسية للتشغيل العالمي:
* **التشغيل بالوضع التفاعلي داخل الطرفية:**
  ```bash
  nova
  ```
* **تشغيل واجهة الويب الاستوديو المتقدمة (NOVA Studio UI):**
  ```bash
  nova ui
  ```
  *(سيقوم هذا الأمر بتشغيل الخادم وفتح المتصفح تلقائياً على لوحة التحكم الرسومية المحدثة).*

* **تخطي رسوم البدء المتحركة (للسرعة):**
  ```bash
  nova --no-animation
  ```

## 📂 2. خريطة العمل على مشروع قائم (Roadmap for Existing Projects)
إذا كان لديك مشروع برمجي جاهز (على سبيل المثال: تطبيق React، أو Node.js API، أو مشروع Python) وتريد أن تقوم منصة NOVA بتحليله، مراجعته، أو إضافة مميزات جديدة إليه، اتبع الخطوات التالية:

### الخطوة الأولى: التوجّه لمجلد المشروع والتحليل
افتح الطرفية داخل مجلد مشروعك القائم ثم نفذ الأمر:
```bash
nova
```
بمجرد دخولك للوضع التفاعلي، قم بتحليل هيكلية وتكوين مشروعك بكتابة:
```
/project
```
سوف تقوم NOVA بقراءة الملفات وتحديد التقنيات المستخدمة (Language, framework, packages) وعرض ملخص كامل للمشروع.

### الخطوة الثانية: كتابة دستور وقواعد المشروع (`NOVA.md`)
لضمان التزام الذكاء الاصطناعي بنفس أسلوبك في كتابة الكود وبنية المجلدات المفضلة لديك، نفذ الأمر:
```
/init
```
سيقوم هذا الأمر بإنشاء ملف `NOVA.md` (Constitution) في مجلد مشروعك. 
> 💡 **نصيحة احترافية:** افتح ملف `NOVA.md` واكتب فيه شروطك الخاصة (مثل: "استخدم مكتبة Tailwind CSS دائماً"، أو "لا تعدل ملفات مجلد config"، أو "اكتب التعليقات باللغة العربية"). ستقوم NOVA بقراءة هذا الملف وتحديث قراراتها البرمجية لتطابق شروطك بدقة 100%.

### الخطوة الثالثة: استخدام واجهة الويب الاستوديو (Studio IDE)
إذا كنت تفضل تعديل الكود من خلال واجهة رسومية فاخرة وجذابة بدلاً من الشاشة السوداء، اكتب داخل المنصة التفاعلية أو مباشرة في الطرفية:
```bash
nova ui
```
ستفتح واجهة الويب **NOVA Studio** المزودة بـ:
* **مستكشف الملفات (Workspace IDE):** تصفح وتعديل وحفظ الملفات بضغطة زر.
* **مقارن التغييرات (Diff Explorer):** رؤية الفرق بين الكود القديم والجديد قبل الحفظ.
* **لوحة نقاش العميل المزدوج (Swarm Debate):** رؤية النقاش التحليلي بين المهندس والمراجع الأمني والمدير قبل البدء بالتعديل.
* **أدوات التفعيل التلقائي (Toolbox):** تشغيل الفحص الأمني بنقرة واحدة.

## 🆕 3. خريطة العمل لإنشاء مشروع جديد من الصفر (Roadmap for New Projects)
إذا كنت تبدأ فكرة مشروع جديد تماماً، فإن NOVA قادرة على توليد خطة العمل وكتابة ملفات التكوين والشيفرات البرمجية وتثبيت المكتبات تلقائياً.

### الخطوة الأولى: إنشاء المجلد وتشغيل المنصة
أنشئ مجلداً فارغاً لمشروعك الجديد وافتح الطرفية بداخله:
```bash
mkdir my-awesome-app
cd my-awesome-app
nova
```

### الخطوة الثانية: توليد خطة التنفيذ البرمجية (Implementation Plan)
قم بتوجيه المنصة لوضع التخطيط لكتابة هيكل المشروع. اكتب الأمر:
```
/plan "إنشاء تطبيق ويب متكامل لإدارة المهام باستخدام React و TypeScript"
```
ستقوم NOVA بتحليل طلبك، وإنشاء ملف خطة عمل تفصيلي باسم `implementation_plan.md` في مجلد مشروعك، يحتوي على:
* هيكل المجلدات المقترح.
* قائمة المكونات والوظائف البرمجية المطلوبة.
* خطوات التنفيذ خطوة بخطوة.

### الخطوة الثالثة: تشغيل وضع العميل المستقل (Agent Mode)
لجعل المنصة تقوم بكتابة الأكواد والملفات المقترحة في الخطة بشكل مستقل، قم بتغيير الوضع إلى وضع العميل:
```
/agent
```
ثم اطلب منه: "نفذ الخطوة رقم 1 ورقم 2 في خطة التنفيذ البرمجية". سيبدأ المساعد باستخدام أداة `file_write` لإنشاء الملفات وملئها بالكود اللازم تلقائياً، وتثبيت التبعيات المطلوبة من خلال تنفيذ الأوامر الآمنة.

## 🛠️ 4. لوحة الأدوات وتعديل الكود تلقائياً (Orchestration & Editing)
من أهم مميزات NOVA هي قدرتها على قراءة وتعديل الكود بدقة عالية دون الحاجة للنسخ واللصق اليدوي.

### الأوامر التفاعلية البرمجية:
| الأمر التفاعلي | الغرض والوظيفة البرمجية |
| :--- | :--- |
| **`/test [command]`** | تشغيل اختبارات المشروع تلقائياً (يكتشف إذا كان Jest أو Vitest أو PyTest ويشغل الفحص فورا). |
| **`/review [file]`** | مراجعة الكود المكتوب في ملف معين، أو مراجعة تغييرات Git غير المحفوظة لاكتشاف الأخطاء المنطقية. |
| **`/security [file]`** | إجراء فحص أمني للملف المختار للتأكد من عدم وجود تسريب كلمات مرور أو ثغرات حقن (Injection). |
| **`/edit [file]`** | يفتح الملف في محرر خارجي مريح لك (مثل VS Code أو Notepad) لكتابة الأسطر الطويلة بحرية، وعند إغلاقه يتم سحبه فوراً للمنصة. |

## 💡 5. تلميحات احترافية وحل المشكلات (Pro-Tips & Troubleshooting)

### 1. الكتابة متعددة الأسطر (Multi-line Prompts):
إذا كنت تريد شرح مشكلة طويلة للذكاء الاصطناعي تحتوي على كود أو تفاصيل معقدة، لا ترسل سطراً بسطر. استخدم إشارة ثلاث علامات اقتباس `"""` للبدء والإنهاء:
```text
"""
مرحباً نوفا، لدي مشكلة في دالة الاتصال بقاعدة البيانات. 
إليك الكود المستخدم:
[اكتب كودك هنا]
كيف يمكنني إصلاح هذا الخطأ؟
"""
```

### 2. تخطي مشكلة حظر المنافذ (Port Blockers):
بشكل افتراضي، تعمل واجهة الويب على منفذ `3141`. إذا كان هذا المنفذ مستخدماً بواسطة عملية أخرى في جهازك، سيظهر خطأ اتصال. يمكنك تشغيل الاستوديو على منفذ بديل بسهولة:
```bash
nova ui --port 3142
```

### 3. ضغط سياق المحادثة وتوفير التوكنز (`/compress`):
إذا طالت المحادثة وبدأ استهلاك الذاكرة يقترب من الحد الأقصى للموديل، اكتب أمر `/compress`. ستقوم NOVA بتلخيص ما تم نقاشه ودمجه في الذاكرة وحذف الرسائل الطويلة القديمة، مما يوفر **60-70%** من حجم التوكنز المتبقية في المحادثة ويحافظ على سرعة الاستجابة.

### 4. اختيار وتغيير موديل الذكاء الاصطناعي محلياً:
يمكنك معرفة الموديلات المتوفرة في برنامج Ollama بجهازك بكتابة:
```
/models
```
ثم التبديل لموديل آخر أكثر سرعة أو أعلى كفاءة (مثل qwen2.5-coder أو llama3) أثناء المحادثة بكتابة:
```
/model qwen2.5-coder:7b
```
